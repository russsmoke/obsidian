# Description
Функция ресурсной эффективности вычислительного алгоритма, учитывающую требования алгоритма к ресурсам компьютера. К таким ресурсам можно отнести:
1) **ресурсы оперативной памяти**, необходимые для хранения исходных данных и т.д. (глобальные и статик переменные, у которых адреса статичные и не меняются). Все локальные переменные внутри функции их память расходуется из стека, программа может предугадать затрачиваемую память, которую они занимают, если это **не рекурсия**. 
2) **ресурсы машинного кода программы**
3) **ресурс стека** при чрезмерном вызове рекурсивной функции, не съедает всю оперативную память, а лишь которая была "запасена" для выполнения программы.
4) **ресурс процессора**
Базовый размер стековой памяти при выполнения программы, но можно размер увеличить: 2 МБ. Большинство программ умещается в такой размер. Но компилятор может увеличить размер стековой памяти.
### Дерево рекурсии

Корень располагается строго сверху.

Факториал это рекурсия первого порядка, числа Фибоначчи это рекурсия второго порядка. 
При выполнении рекурсивных вызовов, эти вызовы хранятся в стеке в виде адреса на функцию. 

### Математические методы анализа рекурсивных алгоритмов

## Графы. Основные понятия
Граф G = (V,E) состоит из множества вершин V и множества дуг Е. Вершины называют узлами, а дуги - ребрами. Графы используются для моделирования отношений между объектами. **Объектами** являются вершинами графа, ребра - отношения между объектами.
